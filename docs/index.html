What this add-on actually does (and why it's hard)

Re-importing an updated .apkg into stock Anki overwrites every field — wiping the personal notes people keep on cards — and a reworded question silently loses its scheduling and starts over as a new card. This add-on fixes both, with nothing but a GitHub repo (or folder) as the "server":

  • A backup first, always. Every sync starts with a timestamped backup of the deck (try Advanced → Import intern pearls deck here to actually roll one back).
  • History survives edits. Cards match by a stable GUID first, then by front text, so a reworded question keeps your intervals.
  • Your annotations survive too. Preserved fields (configurable in Manage decks) are snapshotted before the import and restored after it.
  • Appearance changes ask first. A changed card template means a one-time full AnkiWeb sync, so it's applied only with your consent — and never by the unattended auto-sync (turn it on in Settings and ship an update to see; demo minutes run at 5 seconds).
  • Only what changed. A version hash per deck means editing one deck doesn't re-import the other.

Install the add-on, open Manage decks → Configure source → Try the example deck, and you can do all of this inside real Anki in about a minute. Hosting your own decks is a manifest.json plus .apkg files in any repo — fork the example deck source to start.

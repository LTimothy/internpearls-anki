What just happened (and why it's hard)

Re-importing an updated .apkg into stock Anki overwrites every field — wiping the personal notes people keep on cards — and a reworded question silently loses its scheduling and starts over as a new card. This add-on fixes both, with nothing but a GitHub repo (or folder) as the "server":

  • A backup first, always. Every sync starts with a timestamped backup of the deck, so everything is reversible.
  • History survives edits. Cards match by a stable GUID first (so even a reworded question keeps your intervals and ease), then by front text for older collections.
  • Your annotations survive too. Configurable protected fields are snapshotted before the import and restored after it.
  • Appearance changes ask first. A changed card template means a one-time full AnkiWeb sync, so it's applied only with your consent — never silently, and never by the unattended auto-sync.
  • Only what changed. A version hash per deck means editing one deck doesn't re-import ten.

Install the add-on, open Manage decks → Configure source → Try the example deck, and you can run the real version of this demo inside Anki in about a minute. Hosting your own decks is a manifest.json plus .apkg files in any repo — fork the example deck source to start.

This page is a self-contained simulation — no Anki, no add-on, no data leaves your browser. MIT licensed, like the add-on itself.
